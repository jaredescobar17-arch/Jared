# LinkedIn posts (best-effort)

**Access note.** LinkedIn serves post content only to logged-in sessions and blocks
automated/anonymous fetching (every public post URL we requested returned an empty
login wall). We did **not** scrape LinkedIn. Instead, each file below documents, per
expert, their verified LinkedIn profile plus specific recent posts (URL + date +
a short summary drawn from public search snippets and the post's own headline).

For several of these experts the *verifiable full text* of their thinking lives in
formats we could capture in full — newsletters, blog posts, and podcast/YouTube
transcripts — which are stored in `../youtube-transcripts/` and `../other/`. The
LinkedIn entries here are pointers and context, not a substitute for that primary text.

To collect full post bodies, open the URLs while logged in, or use the official
LinkedIn API / a compliant export.
